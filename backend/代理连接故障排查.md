# 代理连接故障排查

## 当前状态

根据诊断结果:
- ✅ 代理端口 7897 **已开放**
- ✅ DNS 解析正常
- ❌ 但仍然出现连接错误: `Cannot connect to host www.okx.com:443`

## 问题分析

虽然代理端口开放,但请求仍然失败。可能的原因:

### 1. **代理配置问题** (最可能)
- aiohttp 使用代理的方式可能有问题
- SSL 证书验证可能失败

### 2. **代理软件配置问题**
- 代理软件可能不支持 HTTPS
- 代理规则可能阻止了 www.okx.com

## 解决方案

### 方案 A: 修改代理配置 (推荐)

#### 步骤 1: 检查代理软件设置

**Clash 用户:**
1. 打开 Clash 控制面板
2. 确认"系统代理"已开启
3. 检查"规则"或"代理规则",确保 www.okx.com 走代理
4. 尝试在 Clash 中添加规则: `DOMAIN-SUFFIX,okx.com,PROXY`

**V2Ray 用户:**
1. 确认 V2Ray 正在运行
2. 检查出站规则,确保 okx.com 走代理
3. 确认 HTTP 代理端口设置正确

#### 步骤 2: 测试代理是否工作

```bash
# Windows PowerShell
curl.exe -x http://127.0.0.1:7897 https://www.okx.com/api/v5/market/ticker?instId=BTC-USDT
```

如果上面的命令返回 JSON 数据,说明代理工作正常。

#### 步骤 3: 重启后端服务

```bash
# 停止当前后端 (Ctrl+C)
cd F:\Cluade Code Project\OKK\backend
python -m app.main
```

### 方案 B: 临时禁用代理,使用直连

如果你的网络环境可以直接访问 OKX,可以临时禁用代理:

编辑 `backend/app/core/config.py`:
```python
OKX_PROXY: Optional[str] = None  # 禁用代理
```

重启后端服务。

---

### 方案 C: 使用环境变量配置代理

创建 `backend/.env` 文件:
```env
OKX_API_KEY=your_api_key
OKX_SECRET_KEY=your_secret_key
OKX_PASSPHRASE=your_passphrase
OKX_SIMULATED=false
OKX_PROXY=http://127.0.0.1:7897
```

然后修改 `config.py`:
```python
OKX_PROXY: Optional[str] = None  # 从.env读取
```

---

### 方案 D: 使用 SOCKS5 代理(如果支持)

有些代理软件同时提供 HTTP 和 SOCKS5 代理。尝试使用 SOCKS5:

```python
# 在 config.py 中
OKX_PROXY: Optional[str] = "socks5://127.0.0.1:7891"  # Clash SOCKS5 默认端口
```

**注意**: 使用 SOCKS5 需要安装额外的包:
```bash
pip install aiohttp-socks
```

然后修改代码以支持 SOCKS5 代理。

## 常见错误和解决方案

### 错误 1: `Cannot connect to host`
**原因**: 代理未生效或代理配置错误
**解决**:
1. 检查代理软件是否运行
2. 确认代理端口正确
3. 尝试重启代理软件

### 错误 2: `SSLError`
**原因**: SSL 证书验证失败
**解决**: 已在代码中设置 `ssl=False`

### 错误 3: `ProxyConnectionError`
**原因**: 代理拒绝连接
**解决**:
1. 检查代理软件规则设置
2. 尝试在代理软件中添加 okx.com 到代理列表
3. 检查代理认证(如果需要)
