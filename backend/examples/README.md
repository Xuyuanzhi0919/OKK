# 回测示例脚本使用说明

## 快速开始

### 1. 确保后端服务正在运行

```bash
cd backend
python -m app.main
```

后端应该在 `http://localhost:8000` 运行

### 2. 运行回测示例

```bash
cd backend/examples
python run_backtest.py
```

## 脚本功能

`run_backtest.py` 演示了完整的回测流程：

1. **下载K线数据** - 从OKX获取BTC-USDT的1小时K线（最近30天）
2. **运行网格回测** - 使用预设参数执行网格策略回测
3. **分析结果** - 展示详细的性能指标和交易统计
4. **给出建议** - 根据回测表现提供优化建议

## 自定义参数

编辑 `run_backtest.py` 中的参数：

```python
# 在 main() 函数中
SYMBOL = "BTC-USDT"       # 交易对
INTERVAL = "1H"            # K线周期: 1m, 5m, 15m, 30m, 1H, 4H, 1D
DAYS = 30                  # 回测天数
INITIAL_CAPITAL = 10000    # 初始资金 (USDT)
GRID_NUM = 20              # 网格数量
PRICE_LOWER = 60000        # 价格下限
PRICE_UPPER = 70000        # 价格上限
AMOUNT_PER_GRID = 500      # 每格投入金额
```

## 性能指标说明

- **总收益率**: 总盈亏百分比
- **年化收益率**: 按年化计算的收益率
- **夏普比率**: 风险调整后收益（>1.0较好，>2.0优秀）
- **最大回撤**: 权益曲线最大跌幅（越小越好）
- **胜率**: 盈利交易占比

## 常见问题

### Q: 数据下载失败？

检查：
1. 后端服务是否正常运行
2. OKX API代理配置（`backend/.env` 中的 `OKX_PROXY`）
3. 网络连接是否正常

### Q: 回测失败？

检查：
1. K线数据是否下载成功
2. 策略参数是否合理（价格区间要覆盖实际价格）
3. 数据库连接是否正常

### Q: 如何修改时间范围？

修改 `DAYS` 参数：
```python
DAYS = 60  # 回测最近60天
```

### Q: 支持哪些K线周期？

- 分钟: `1m`, `5m`, `15m`, `30m`
- 小时: `1H`, `4H`
- 日: `1D`
