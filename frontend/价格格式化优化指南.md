# 前端价格格式化优化指南

## 概述

本文档记录了前端价格显示和输入的统一优化方案，确保整个应用的价格格式化保持一致性和专业性。

## 优化内容

### 1. 增强的 `format.ts` 工具函数

已在 `frontend/src/utils/format.ts` 中添加了以下新的格式化函数：

#### 价格格式化
- `formatPrice(price)` - 智能格式化价格（带千位分隔符）
- `formatPriceDisplay(price)` - 价格显示（固定精度，移除末尾零）

#### 金额格式化 (USDT)
- `formatAmount(amount)` - 金额格式化（2位小数，移除末尾零）
- `formatAmountDisplay(amount)` - 金额显示（带千位分隔符）

#### 数量格式化
- `formatQuantity(quantity)` - 数量格式化（智能精度）
- `formatQuantityDisplay(quantity)` - 数量显示（8位小数，移除末尾零）

#### 百分比格式化
- `formatPercent(percent, decimals)` - 百分比格式化（不含%）
- `formatPercentDisplay(percent, decimals)` - 百分比显示（含%）

#### 手续费格式化
- `formatFeeDisplay(fee, decimals)` - 手续费显示（智能精度）

### 2. 统一标准

| 数据类型 | 精度标准 | 推荐函数 | 示例 |
|---------|---------|---------|------|
| 价格 (Price) | 根据大小：100+ → 2位, 1-100 → 4位, 0.01-1 → 6位, <0.01 → 8位 | `formatPriceDisplay()` | 0.20795 → "0.20795" |
| 金额 (USDT) | 2位小数，移除末尾零 | `formatAmount()` | 166.64 → "166.64" |
| 金额显示（大额） | 2位小数 + 千位分隔符 | `formatAmountDisplay()` | 12345.67 → "12,345.67" |
| 数量 (Quantity) | 8位小数，移除末尾零 | `formatQuantityDisplay()` | 801.29746 → "801.2975" |
| 百分比 | 1-2位小数 | `formatPercentDisplay()` | 50.123 → "50.12%" |
| 手续费 | 4-6位小数，智能调整 | `formatFeeDisplay()` | 0.641037968 → "0.641" |

## 替换步骤

### 自动化替换方案

可以使用 VS Code 的全局搜索替换功能：

1. **搜索模式：** `\.toFixed\((\d+)\)`
2. **根据上下文选择对应的格式化函数替换**

### 批量替换注意事项

1. **添加导入语句**
   ```typescript
   import { formatPriceDisplay, formatAmount, formatQuantityDisplay, formatPercentDisplay, formatFeeDisplay } from '@/utils/format'
   ```

2. **移除自定义格式化逻辑**
   - 删除组件内部的自定义 formatPrice 函数
   - 统一使用 utils/format.ts 中的函数

3. **调整参数**
   - `toFixed(2)` → `formatAmount()` 或 `formatPriceDisplay()`
   - `toFixed(8)` → `formatQuantityDisplay()`
   - `(value * 100).toFixed(2)%` → `formatPercentDisplay(value * 100)`

4. **处理空值**
   - 新的格式化函数已内置空值处理
   - 返回 `'--'` 而不是报错

## 优势

1. **统一标准** - 所有价格格式化遵循相同规则
2. **智能精度** - 根据数值大小自动调整小数位数
3. **移除末尾零** - 自动清理无意义的末尾零
4. **健壮性** - 内置空值和错误处理
5. **可维护性** - 集中管理，易于修改
