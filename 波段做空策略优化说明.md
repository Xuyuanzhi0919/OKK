## 优化内容概览

### 1. ✅ 修复post_only订单问题（核心修复）

**问题描述**：
- 原代码使用`post_only`订单类型，该类型只能作为maker挂单，不能立即成交
- 当市场价格快速波动时，订单可能长时间无法成交，导致错过最佳交易时机

**优化方案**：
```python
# 修改前：
order_type_str, order_price = "post_only", limit_price

# 修改后：
order_type_str = "ioc"  # immediate_or_cancel
```

**优化效果**：
- ✅ IOC订单可以立即成交（taker）或部分成交，未成交部分自动撤销
- ✅ 既保留了争取更好手续费率的优势，又确保订单能够成交
- ✅ 解决了"订单挂着无法成交"的问题

### 4. ✅ 新增追踪止损功能（Trailing Stop）

**功能说明**：
追踪止损是一种动态止盈方式，当价格朝有利方向发展时，自动调整止盈位，锁定更多利润。

**参数配置**：
```json
{
  "enable_trailing_stop": true,  // 启用追踪止损
  "trailing_stop_pct": 2,        // 追踪止损回调比例2%
  "trailing_stop_activation_pct": 3  // 盈利3%后激活追踪止损
}
```

**工作原理**：
1. **激活条件**：当保证金盈利达到`trailing_stop_activation_pct`（如3%）时，激活追踪止损
2. **追踪逻辑**：
   - 做空策略：价格持续下跌时，更新最低价（最有利价格）
   - 当价格从最低点反弹超过`trailing_stop_pct`（如2%）时，触发平仓
3. **示例**：
   ```
   开空仓价：2200
   价格跌到2133（盈利3%）→ 激活追踪止损
   价格继续跌到2100（盈利4.5%）→ 更新追踪基准价为2100
   价格反弹到2142（从2100反弹2%）→ 触发追踪止盈，锁定4.5%利润
   ```

**优势**：
- ✅ 自动锁定利润，避免盈利回吐
- ✅ 允许价格朝有利方向继续发展，不轻易止盈
- ✅ 价格反转时及时平仓，保护已获利润

## 常见问题

### Q1: 为什么选择IOC而不是post_only？
**A**: IOC订单可以立即成交，既保留了maker费率优势，又避免了post_only无法成交的问题。

### Q2: 追踪止损适合什么场景？
**A**: 适合趋势性行情，当价格朝有利方向持续发展时，自动锁定更多利润。不适合震荡行情。

### Q3: 如何关闭追踪止损？
**A**: 设置参数 `"enable_trailing_stop": false`

### Q4: 止盈比例是如何计算的？
**A**: 止盈比例 = 币价变化率 × 杠杆倍数
- 例如：5倍杠杆，币价跌3% → 保证金盈利15% → 触发止盈

### Q5: 当前未成交的订单怎么办？
**A**:
1. 手动撤销（推荐）
2. 或重启策略，策略会自动撤销未成交订单
3. 或等待价格回调（可能需要很长时间）

## 风险提示

⚠️ **重要提示**：
